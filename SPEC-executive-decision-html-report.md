# SPEC: reporte ejecutivo HTML para decisión ID19

## Objetivo

Crear un reporte HTML comprensible para un decisor ejecutivo sobre el análisis de brecha ID19 por CSMC.

## Audiencia

Directivo o equipo de gestión que necesita decidir si usa el análisis como insumo operativo, argumento de negociación o base para una siguiente validación.

## Mensaje crítico

Con la meta vigente, solo 99 de 211 CSMC modelados lograrían el ID19, equivalente a 46.9%. El reporte debe explicar esa brecha de cumplimiento y orientar la focalización operativa.

## Contenido requerido

- Problema.
- Pregunta.
- Por qué es importante.
- Qué se hizo.
- Resultados.
- Recomendaciones.
- Desplegables por región con CSMC focalizados, RENAES y paquetes adicionales.
- Sensibilidad por número de médicos.
- Sensibilidad por número de psicólogos.
- Cinco preguntas MCQ interactivas al final.

## Criterios de aceptación

- El reporte es un HTML autónomo.
- El lenguaje no requiere formación estadística.
- La decisión recomendada queda explícita.
- Las limitaciones críticas aparecen antes de los detalles técnicos.
- Las cinco MCQ quedan al final del HTML y permiten marcar una opción.
- Cada MCQ muestra feedback inmediato de correcto o incorrecto.
- La sección de focalización incluye desplegables por región con los centros priorizados.
- La sección de sensibilidad por médicos declara si cambia o no la recomendación principal.
- La sección de sensibilidad por psicólogos declara si cambia o no la recomendación principal.
